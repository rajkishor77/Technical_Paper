🔷 1. Introduction to Tableau Basics

Tableau is a powerful visual analytics platform used to create interactive data visualizations and dashboards — without writing code. It was founded in 2003 and acquired by Salesforce in 2019.

Why Tableau?

  • Fastest visualization tool — processes large data quickly
  • No programming knowledge required
  • Connects to 50+ data sources (Excel, SQL, Cloud, etc.)
  • Drag-and-drop interface for creating dashboards
  • Supports real-time and live data analysis

✅ Advantages

  • Very fast calculations (backend engine)
  • Interactive & beautiful dashboards
  • No manual calculations needed
  • Handles huge datasets smoothly
  • Strong community & documentation

❌ Disadvantages

  • High cost for Desktop & Server
  • Parameters are static & single-value only
  • Limited data preprocessing features
  • Not ideal for complex ETL tasks
  • Steep learning curve for LOD & advanced calc

Tableau Products

⚙️ 2. Installation & Setup Basics

🖥️ 4. Tableau Interface Basics

Understanding the Tableau interface is key before building anything.

Key Areas

  • Workbook — the main file (.twb or .twbx)
  • Worksheet — single chart/view canvas
  • Dashboard — combine multiple worksheets
  • Story — sequence of dashboards for presentation

Important Panels

  • Data Pane — shows all fields (dimensions + measures)
  • Analytics Pane — add trend lines, reference lines, forecasts
  • Marks Card — controls Color, Size, Label, Detail, Tooltip
  • Show Me — suggests chart types based on selection
📌 Rows & Columns shelf — Drag fields here to build your visualization. Dimensions go on Columns, Measures on Rows (usually).

🔄 5. Data Modeling Important

Joins

Combining two or more tables based on a common field (like SQL joins).

Join TypeReturns
Inner JoinOnly matching rows from both tables
Left JoinAll rows from left table + matching from right
Right JoinAll rows from right table + matching from left
Full Outer JoinAll rows from both tables (nulls where no match)

Relationships (New in Tableau 2020.2+)

Relationships are the modern, smarter way to combine tables. Unlike joins, they work at query time and handle multiple levels of granularity without data duplication. Prefer Relationships over Joins for most use cases.

Union

Stacks rows from multiple tables vertically. Tables must have the same columns. Used when you want to combine data from Jan sheet + Feb sheet, etc.

Data Blending

Joins data from different data sources at the viz level. One is the "primary" and the other is "secondary." Less efficient than joins — use when sources can't be combined at DB level.

🧹 6. Data Cleaning Basics

Done in the Data Source tab before building visualizations.

  • Rename Columns — double-click column header in Data Source
  • Change Data Types — click the icon beside the field name (Abc, #, calendar icon)
  • Hide Fields — right-click → Hide (removes from view, not from data)
  • Split — splits one column into multiple (e.g., "First Last" → "First" + "Last")
  • Pivot — converts wide data to long format (columns become rows)
  • Remove Null Values — use Filters or IFNULL() / ZN() in calculated fields
  • Data Interpreter — Tableau auto-detects and fixes messy Excel formatting

📐 8. Dimensions & Measures Important

Dimensions (Blue)

Categorical / qualitative fields. Used to slice and segment data.

  • Examples: City, Product Name, Category, Region
  • Usually discrete (independent, separate values)
  • Goes on Rows, Columns, Color, Filter

Measures (Green)

Numeric / quantitative fields. Used to aggregate and calculate.

  • Examples: Sales, Profit, Discount, Quantity
  • Usually continuous (form an axis)
  • Tableau auto-aggregates them (SUM, AVG, etc.)

Continuous vs Discrete

Discrete (Blue pill)Continuous (Green pill)
BehaviorCreates headers/labelsCreates axis
ExampleMonth (Jan, Feb, Mar...)Sales axis (0 to 1M)
SwitchRight-click any field → Convert to Discrete / Continuous

Common Aggregations

Right-click a measure → Measure → choose: SUM, AVG, MIN, MAX, COUNT, COUNTD (distinct count), MEDIAN

🗺️ 11. Geographic Visualization

Tableau has built-in geographic recognition. If a field contains Country, State, City, ZIP Code data — Tableau auto-assigns a Geographic Role.

Filled Map

Shades entire regions (countries/states) based on a measure value. Good for showing regional performance.

Symbol Map

Places circles/shapes on map. Size/color represents measure. Better for city-level or point data.

Setting Geographic Roles

Right-click a dimension → Geographic Role → choose Country, State, City, ZIP, Latitude, Longitude

💡 Double-click Latitude + Longitude fields (or any geo field) → Tableau auto-generates a map view!

🔍 12. Sorting & Filtering Must Know

Filters (Order of Operations — Top to Bottom)

  1. Extract Filter — applied when creating extract
  2. Data Source Filter — applied to all sheets in workbook
  3. Context Filter — creates a temp table; other filters run within it
  4. Dimension Filter — filters by dimension values
  5. Measure Filter — filters by aggregated measure values
  6. Table Calculation Filter — applied last (after calculation)
Filter TypeWhen to Use
Quick FilterShow filter control on dashboard for user interaction
Context FilterMake Top N filter work correctly; creates sub-dataset
Top N FilterShow only Top/Bottom N items (needs Context Filter for accuracy)
Conditional FilterFilter based on condition (e.g., SUM(Sales) > 10000)

Sorting

Click the sort icon on axis, or go to Data → Sort Fields. Options: Alphabetical, By Field (measure), Manual drag-sort, or Data Source order.

🗂️ 13. Groups, Sets & Hierarchies Important

Groups

Manually combine dimension members into a new category.

Example: Group "East" + "West" → "Coastal Regions"

Create: Select members → right-click → Group

Sets

Dynamic subsets based on a condition or manual selection. Creates In/Out classification.

Example: Customers with Sales > $5000 = "Top Customers Set"

Hierarchies

Organize fields in a drill-down structure.

Example: Country → State → City → ZIP

Create: Drag one field onto another in the Data pane → Name the hierarchy

Drill Down

Click the + icon on a hierarchy field in the view to expand to the next level. Click − to collapse back up.

🧮 14. Calculated Fields Must Know

Create new fields using formulas. Right-click in Data pane → Create Calculated Field

IF / ELSE

IF [Sales] > 10000 THEN "High" ELSEIF [Sales] > 5000 THEN "Medium" ELSE "Low" END

CASE Statement

CASE [Region] WHEN "East" THEN "Eastern" WHEN "West" THEN "Western" ELSE "Other" END

String Functions

FunctionDescription
LEN([Name])Length of string
LEFT([Name], 3)First 3 characters
UPPER([Name])Convert to uppercase
TRIM([Name])Remove leading/trailing spaces
CONTAINS([Name], "abc")Check if string contains value
REPLACE([Name], "old", "new")Replace text

Date Functions

FunctionExample

DATEDIFF('day', [Start], [End])Days between two dates

Null Handling

  • ZN([Field]) — shortcut for IFNULL([Field], 0)
  • ISNULL([Field]) — returns True if null

📉 15. Table Calculations Important

Calculations performed on the result set already returned (not on the raw data). Applied in the view level.

Add: Right-click a measure in view → Quick Table Calculation

CalculationWhat it Does
Running TotalCumulative sum up to current row
Moving AverageAverage over N preceding periods (smooths trends)
Percent of TotalEach value as % of grand total
Percent Difference% change from one period to next
RankRanks each value (1 = highest by default)
Window FunctionsWINDOW_SUM, WINDOW_AVG, WINDOW_MAX, etc.
💡 Compute Using matters! Right-click → Edit Table Calculation → choose: Table (Across), Table (Down), Pane, Cell, or a specific dimension.

🎛️ 16. Parameters Important

Parameters are dynamic input controls that users can change — they power interactive dashboards.

Create: Right-click in Data pane → Create Parameter

How Parameters Work

  1. Create a Parameter (set type: integer, string, float, list)
  2. Show Parameter control (right-click → Show Parameter)
  3. Reference it in a Calculated Field using [Parameter Name]
  4. Use the calculated field in your viz

Common Use Cases

  • Dynamic Top N filter (user selects N = 5, 10, 20...)
  • Switch between measures (show Sales vs Profit)
  • Dynamic date ranges
  • Reference line threshold (set profit goal)
  • What-if analysis (growth rate slider)

🔬 17. Level of Detail (LOD) Expressions Very Important

LOD expressions let you control the granularity of a calculation — independently of what's in the view. This is a frequently tested topic in interviews.

TypeSyntaxWhen to Use
FIXED{FIXED [Dim] : AGG([Measure])}Calculate at a specific level regardless of view
INCLUDE{INCLUDE [Dim] : AGG([Measure])}Add extra dimension to view's granularity
EXCLUDE{EXCLUDE [Dim] : AGG([Measure])}Remove a dimension from view's calculation

Practical Examples

  • {FIXED [Customer ID] : SUM([Sales])} → Total sales per customer (even if view is at order level)
  • {FIXED [Region] : AVG([Sales])} → Average sales per region (used in comparison)
  • {EXCLUDE [Month] : SUM([Sales])} → Annual total even when Month is in view
⚠️ FIXED vs Context Filter: FIXED LOD ignores dimension filters but respects Context Filters. Always add your filter to Context if you want LOD to respect it.

🖼️ 18. Dashboard Creation Must Know

A Dashboard combines multiple worksheets + objects into a single interactive view.

Dashboard Objects

  • Vertical / Horizontal Containers — for layout management (tiled vs floating)
  • Text Box — titles, labels, annotations
  • Image — logos, backgrounds
  • Blank — create spacing
  • Web Page — embed external URLs
  • Extension — third-party add-ons

Layout Tips

  • Use Tiled layout for structured, grid-based designs
  • Use Floating for overlapping elements and creative layouts
  • Set dashboard size: Fixed (specific px) or Automatic (responsive)
  • Use Device Preview to design for Phone, Tablet, Desktop separately

⚡ 19. Dashboard Actions Important

Actions make dashboards interactive. Go to Dashboard → Actions → Add Action

Action TypeWhat it Does
Filter ActionClicking a mark filters another sheet. Most common!
Highlight ActionClicking a mark highlights related marks in other sheets
URL ActionClicking opens a URL (link to website, Google Maps, etc.)
Parameter ActionUser click changes a parameter value dynamically
Set ActionUser click adds/removes items from a Set
💡 Filter Action setup: Source Sheet → Target Sheets → Run Action on: Hover / Select / Menu. "Clearing the selection" options control what happens when nothing is selected.

🎨 20. Formatting & UI Design

Good formatting = professional dashboards. Access via Format menu.

  • Fonts — Format → Font → set for whole workbook or specific sheet
  • Colors — use company brand colors; drag a dimension to Color mark
  • Tooltips — customize with text, images, and even Viz in Tooltip (embed a mini-chart!)
  • Borders/Shading — Format → Borders / Shading for table formatting
  • Labels — click Label mark card → show marks labels, position, overlap control
  • Null values — Format → Special Values → choose how nulls appear (default, gap, zero)
🎨 Best Practice: Use max 2–3 colors per dashboard. One highlight color for key metrics. Gray for background/neutral elements.

📖 21. Storytelling with Tableau

A Story is a sequence of Story Points — each point is a snapshot of a dashboard or sheet with a narrative caption. Used for presentations and business reports.

  1. Create your dashboards/sheets
  2. Go to New Story tab
  3. Drag dashboards into Story Points
  4. Add caption text to each point
  5. Publish or present in presentation mode

🗄️ 23. SQL + Tableau Integration Must Know for Jobs

Most data analyst roles require SQL knowledge alongside Tableau. You'll connect Tableau to a SQL database and sometimes write custom SQL queries.

Connecting to MySQL / PostgreSQL

  1. On Start Page → Under "To a Server" → click MySQL / PostgreSQL
  2. Enter Server hostname, Port, Database name, Username, Password
  3. Click Sign In → Tableau shows all tables
  4. Drag tables to canvas to join them

Custom SQL in Tableau

Instead of using tables directly, you can write your own SQL: In Data Source tab → New Custom SQL → Write your SELECT query.

SELECT order_id, customer_name, SUM(sales) as total_sales FROM orders WHERE order_date >= '2023-01-01' GROUP BY order_id, customer_name
💡 Use Custom SQL when you need pre-aggregated data, complex joins, or filtered data that Tableau's visual interface can't handle efficiently.

🚀 24. Advanced Tableau Topics Job-Ready Skills

Forecasting

Analytics Pane → Drag Forecast to view. Works on time-series data. Uses exponential smoothing. Right-click → Forecast Options to customize.

Trend Lines

Analytics Pane → Drag Trend Line. Types: Linear, Logarithmic, Exponential, Power, Polynomial. Right-click → Describe Trend Line to see R² value.

Clustering

Analytics Pane → Drag Cluster on scatter plot. Tableau uses K-Means. Right-click cluster → Describe Clusters for details.

Reference Lines & Bands

Analytics Pane → Reference Line (single value line, e.g., avg), Reference Band (shaded region between two values), Reference Distribution (percentiles).

Performance Optimization

  • Use Extract instead of Live
  • Reduce marks in view
  • Use Context Filters
  • Avoid nested LOD calculations
  • Use Performance Recording: Help → Settings → Start Performance Recording

Row-Level Security

Restrict data access per user. Use User Functions: USERNAME(), ISMEMBEROF(). Typically implemented on Tableau Server/Cloud with User Filters.
